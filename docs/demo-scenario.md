# 룰북 심판관 — 데모 시나리오 (1분)

## 사전 준비

| 항목 | 내용 |
|------|------|
| 데모용 룰북 | 10페이지 이내의 보드게임 룰북 PDF (예: 스플렌더, 아줄, 카탄 등) |
| 환경 | `docker compose up` 으로 백엔드 + 프론트엔드 실행 완료 |
| 브라우저 | Chrome, 개발자 도구 닫힌 상태, 전체화면 권장 |
| 초기 상태 | 세션 없음 (localStorage 초기화 또는 시크릿 모드) |
| 언어 | 한국어 (기본값) |

---

## 데모 흐름

### Scene 1 — 빈 상태 + 룰북 업로드 (0:00 ~ 0:15)

**화면:** 앱 초기 화면 (빈 상태)

| 시간 | 액션 | 화면 변화 |
|------|------|-----------|
| 0:00 | 앱 첫 화면 노출 | 사이드바: "룰북을 업로드하세요" 안내, 채팅 영역: 빈 상태 일러스트, 입력창 비활성 |
| 0:03 | 사이드바 "룰북 업로드" 버튼 클릭 | 업로드 모달 표시 |
| 0:05 | 준비한 PDF를 드래그앤드롭 | 파일명 + 크기 미리보기 |
| 0:07 | "업로드" 버튼 클릭 | 프로그레스 스피너 표시 |
| 0:12 | 처리 완료 (텍스트 추출 → 청킹 → 임베딩 → 인덱싱) | 모달 자동 닫힘, 사이드바에 룰북 제목 / 페이지 수 / 청크 수 표시 |

> **포인트:** PDF 한 장이면 끝 — 별도 설정 없이 즉시 질문 가능 상태가 된다.

---

### Scene 2 — 규칙 질문 + 인용 답변 (0:15 ~ 0:35)

**화면:** 채팅 영역 활성화, 입력창 사용 가능

| 시간 | 액션 | 화면 변화 |
|------|------|-----------|
| 0:15 | 채팅 입력창에 질문 입력: **"게임 준비는 어떻게 하나요?"** | 텍스트 입력 |
| 0:18 | 전송 (Ctrl+Enter 또는 전송 버튼) | 사용자 메시지 즉시 표시 (파란 말풍선), 타이핑 인디케이터 점 세 개 |
| 0:25 | 응답 수신 | 마크다운 답변 + `[p.3, §게임 준비]` 형태의 **인용 배지** 인라인 표시 |

> **포인트:** 답변 안에 출처가 바로 붙어 있다 — "이거 맞아?" 논쟁 끝.

---

### Scene 3 — 인용 출처 검증 (0:35 ~ 0:45)

**화면:** 인용 배지가 포함된 답변이 표시된 상태

| 시간 | 액션 | 화면 변화 |
|------|------|-----------|
| 0:35 | 답변 내 `[p.3, §게임 준비]` 인용 배지 클릭 | 팝오버 열림, 로딩 스피너 |
| 0:37 | 소스 데이터 수신 | 페이지 번호, 섹션명, **원문 텍스트** 표시 + 관련 문장 하이라이트 |
| 0:42 | 팝오버 닫기 (외부 클릭 또는 Esc) | 팝오버 닫힘, 포커스 복귀 |

> **포인트:** 원문을 직접 확인할 수 있다 — AI가 지어낸 게 아니라 룰북에 실제로 있는 내용.

---

### Scene 4 — 프리셋 전환 + 톤 차이 (0:45 ~ 0:55)

**화면:** 채팅 하단 프리셋 칩 영역

| 시간 | 액션 | 화면 변화 |
|------|------|-----------|
| 0:45 | 프리셋 칩에서 **"학습"** 모드 클릭 (기본: 심판) | 칩 활성 상태 변경, 서버에 프리셋 업데이트 |
| 0:48 | 후속 질문 입력: **"좀 더 쉽게 설명해줘"** | 전송 |
| 0:53 | 응답 수신 | 같은 규칙을 **초보자 눈높이로 재설명** — 단계별 분리, 친절한 톤, "다음에 알면 좋은 것" 추천 |

> **포인트:** 같은 룰북, 같은 질문이라도 모드에 따라 답변 스타일이 달라진다 — 초보 학습 / 셋업 가이드 / 분쟁 판정.

---

### Scene 5 — 마무리 (0:55 ~ 1:00)

| 시간 | 액션 | 화면 변화 |
|------|------|-----------|
| 0:55 | 헤더 언어 토글 클릭 (한국어 → EN) | 모든 UI 텍스트 영어 전환 |
| 0:58 | 전체 화면 노출 | 영문 UI + 대화 기록 유지 |

> **포인트:** 한국어 / 영어 양방향 지원 — UI와 AI 답변 모두.

---

## 데모 핵심 메시지 (1문장)

> **보드게임 룰북 PDF를 올리면, AI가 원문 인용과 함께 규칙을 설명해주는 RAG 기반 Q&A 앱.**

## 데모에서 보여주는 기능 요약

| # | 기능 | 대응 시나리오 | 시간 |
|---|------|---------------|------|
| 1 | PDF 업로드 → 자동 인덱싱 | US-01 | 0:00 ~ 0:15 |
| 2 | 자연어 질문 → 인용 포함 답변 | US-02 | 0:15 ~ 0:35 |
| 3 | 인용 클릭 → 원문 검증 | US-03 | 0:35 ~ 0:45 |
| 4 | 프리셋 전환 (심판 → 학습) | US-05 연관 | 0:45 ~ 0:55 |
| 5 | 언어 전환 (한 → 영) | US-06 | 0:55 ~ 1:00 |

## 데모에서 생략하는 기능

| 기능 | 사유 |
|------|------|
| 멀티턴 컨텍스트 유지 (US-04) | Scene 4에서 후속 질문으로 암시적으로 포함 |
| 모델 변경 (US-05) | 시간 제약, 프리셋 전환으로 차별화 충분 |
| 새 세션 시작 (US-07) | 시간 제약 |
| 빈 상태 상세 (US-08) | Scene 1에서 3초간 노출로 충분 |
