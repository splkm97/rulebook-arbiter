# 룰북 심판관 — 사용자 시나리오

## 개요

보드게임 룰북 PDF를 업로드하면 RAG(ChromaDB + Gemini) 기반으로 규칙에 대한 질의응답을 제공하는 웹 애플리케이션.

---

## US-01: 룰북 업로드

**선행조건:** 사용자가 앱에 접속한 상태
**주 행위자:** 사용자

| 단계 | 행위 | 기대 결과 |
|------|------|-----------|
| 1 | 사이드바의 "룰북 업로드" 버튼 클릭 | 업로드 모달 표시 |
| 2 | PDF 파일을 드래그앤드롭 또는 클릭으로 선택 | 파일명과 크기 표시 |
| 3 | "업로드" 버튼 클릭 | 처리 중 프로그레스 표시 |
| 4 | 서버에서 PDF 텍스트 추출 → 청킹 → 임베딩 → 인덱싱 완료 | 모달 자동 닫힘, 사이드바에 룰북 제목/페이지/청크 수 표시 |

**예외 흐름:**

- **E1:** PDF가 아닌 파일 업로드 → 400 에러, "유효한 PDF 파일이 아닙니다" 메시지
- **E2:** 스캔된 이미지 PDF (텍스트 없음) → 400 에러, "텍스트 추출 불가" 메시지
- **E3:** 50MB 초과 파일 → 프론트엔드에서 차단, 업로드 불가
- **E4:** 네트워크 오류 → 에러 메시지 표시, 재시도 가능

---

## US-02: 규칙 질문 (채팅)

**선행조건:** 룰북이 업로드된 세션 존재
**주 행위자:** 사용자

| 단계 | 행위 | 기대 결과 |
|------|------|-----------|
| 1 | 채팅 입력창에 질문 입력 (예: "게임 준비 방법은?") | 텍스트 입력 가능 |
| 2 | Ctrl+Enter 또는 전송 버튼 클릭 | 사용자 메시지가 채팅에 즉시 표시 (낙관적 업데이트) |
| 3 | 서버에서 RAG 파이프라인 실행 | 타이핑 인디케이터 표시 |
| 4 | 응답 수신 | `[p.X, §Y]` 형태의 인라인 인용이 포함된 마크다운 답변 표시 |

**예외 흐름:**

- **E1:** 세션 만료/없음 → 404 에러, "세션을 찾을 수 없습니다" 메시지
- **E2:** 룰북에 없는 내용 질문 → "업로드된 룰북에 해당 정보가 없습니다" 답변
- **E3:** LLM API 오류 → 502 에러 메시지 표시

## US-03: 인용 출처 확인

**선행조건:** 인라인 인용이 포함된 답변이 표시된 상태
**주 행위자:** 사용자

| 단계 | 행위 | 기대 결과 |
|------|------|-----------|
| 1 | 답변 내 `[p.3, §셋업]` 형태의 인용 배지 클릭 | 팝오버 표시, 로딩 스피너 |
| 2 | 서버에서 소스 청크 데이터 수신 | 페이지 번호, 섹션명, 원문 텍스트 표시 |
| 3 | 팝오버 외부 클릭 또는 Escape 키 | 팝오버 닫힘 |

**특이사항:** 인용 데이터는 `staleTime: Infinity`로 캐시되어 같은 인용을 다시 클릭하면 즉시 표시.

---

## US-04: 대화 컨텍스트 유지 (멀티턴)

**선행조건:** 하나 이상의 질문-답변이 완료된 상태
**주 행위자:** 사용자

| 단계 | 행위 | 기대 결과 |
|------|------|-----------|
| 1 | 후속 질문 입력 (예: "그러면 첫 턴에는 무엇을 하나요?") | 이전 대화 컨텍스트가 LLM에 전달 |
| 2 | 응답 수신 | 이전 답변의 맥락을 반영한 답변, 새로운 인용 포함 |

**제약조건:** 대화 기록은 최대 10쌍(20턴)으로 제한. 초과 시 가장 오래된 쌍 삭제.

---

## US-05: 모델 변경

**선행조건:** 활성 세션 존재
**주 행위자:** 사용자

| 단계 | 행위 | 기대 결과 |
|------|------|-----------|
| 1 | 헤더의 모델 선택 드롭다운 클릭 | gemini-2.0-flash, gemini-2.0-pro 옵션 표시 |
| 2 | 다른 모델 선택 | PUT /api/settings 호출, 스토어 업데이트 |
| 3 | 다음 질문 전송 | 새로 선택한 모델로 응답 생성 |

**예외 흐름:**

- **E1:** 유효하지 않은 모델명 → 400 에러

---

## US-06: 언어 전환

**선행조건:** 앱이 로드된 상태 (기본: 한국어)
**주 행위자:** 사용자

| 단계 | 행위 | 기대 결과 |
|------|------|-----------|
| 1 | 헤더의 언어 토글 클릭 (한국어 ↔ EN) | 모든 UI 텍스트가 선택 언어로 전환 |
| 2 | 질문 입력 | LLM이 질문 언어에 맞춰 답변 (시스템 프롬프트 규칙) |

---

## US-07: 새 세션 시작

**선행조건:** 기존 세션이 활성 상태
**주 행위자:** 사용자

| 단계 | 행위 | 기대 결과 |
|------|------|-----------|
| 1 | 사이드바의 "새 세션" 버튼 클릭 | 세션 초기화, 채팅 기록 삭제 |
| 2 | 새 룰북 업로드 | 새로운 세션 ID로 독립적인 세션 생성 |

---

## US-08: 빈 상태 (초기 화면)

**선행조건:** 앱 최초 접속, 세션 없음
**주 행위자:** 사용자

| 상태 | 표시 내용 |
|------|-----------|
| 사이드바 | "룰북을 업로드하세요" 안내 메시지, 업로드 버튼 |
| 채팅 영역 | "룰북을 업로드하고 질문을 시작하세요" 빈 상태 일러스트 |
| 채팅 입력 | 비활성 (disabled), 세션 없이는 입력 불가 |

---

## 비기능 요구사항

| 항목 | 요구 |
|------|------|
| 응답 시간 | PDF 업로드 처리 < 30초 (일반적인 룰북 기준), 채팅 응답 < 10초 |
| 접근성 | 키보드 내비게이션, ARIA 속성, 스크린 리더 지원 |
| 반응형 | 모바일 (사이드바 오버레이), 데스크톱 (고정 사이드바) |
| 다크 모드 | 시스템 설정 감지, Tailwind dark: 클래스 적용 |
| 에러 복원 | 모든 API 호출에 에러 핸들링, 사용자 친화적 메시지 |
